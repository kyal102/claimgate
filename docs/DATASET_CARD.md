# Dataset Card — ProofBench X (Prototype)

## Splits
- **public / dev** — deterministic, seeded cases generated by the benchmark's own generators (default master seed). Used for development and regression. **Not leaderboard-eligible** (see below).
- **private holdout** — fresh cases generated from the same generators with a **new seed after implementation**, with expected answers withheld from any model under test. **The only leaderboard-eligible split.**

Generate a holdout:
```
python -m claimgate run --v1 --holdout --seed <new_seed> --json
```

## Contaminated cases
- Any case an evaluated model (or its builder) has previously seen.
- The **default-seed public/dev cases are treated as contaminated for model leaderboard purposes**, because they were available during development.
- Cases whose generation, selection, or expected answers were produced with AI assistance from a model that is later evaluated.

## AI-assisted cases
This benchmark and its tooling were built with AI assistance. Therefore:
- A model that helped build the benchmark **cannot** post an uncontaminated raw score on the dev cases.
- Such runs must be labeled `CONTAMINATED` / `is_prototype=true`.

## Leaderboard eligibility
| Split | Model raw score | Model+tool score | DTL-verified score |
|---|---|---|---|
| public / dev | not eligible (contaminated) | not eligible | informational only |
| private holdout (post-impl, unseen) | **eligible** | **eligible** | **eligible** |

A DTL-verified score is the verifier's own correctness and is not a "model" score, but it is still only *official* on an uncontaminated holdout.

## Provenance
- Generators: `claimgate/core/families.py`, `core/metamorphic.py`, `core/counterexamples.py`, `core/v2/`, gate-specific bench modules.
- Determinism: every case is a pure function of `(generator, seed)`; same seed ⇒ identical cases.
